# Project setup and planning

## Required files

| File | Purpose |
|---|---|
| `AGENTS.md` | project context, commands, boundaries |
| `docs/architecture.md` | architectural decisions and invariants |
| `CHANGELOG.md` | release history |

Use `assets/AGENTS.md.template` and `assets/plan.md.template` as the starting point.

## plan.md usage

`plan.md` is an ephemeral work file that lives outside the repo and should be deleted after the work merges.

Every plan must include:
- goal
- 3–7 steps
- acceptance criteria
- tests
- key risks / unknowns
- files to touch
